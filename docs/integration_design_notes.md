# 日報システム × 経営管理システム 連携設計メモ

作成日：2026-04-25

---

## 現状の問題点

両システムは現在「並列稼働」であり「連携」していない。
データが繋がっていないため、案件単位の損益（実粗利）が計算できない。

```
【現状】
日報システム  → 稼働表Excel（/日報/）
経営管理システム → 経費管理Excel（/経費管理/）

← この2つは別ファイル・別シートで完全に独立している →
```

---

## 連携で目指すデータフロー

```
LINEで日報送信
    ↓
稼働表に記録（人員・現場・時間・移動費）
    ↓
    ├─→ 請求集計（得意先別）
    └─→ 経営管理システムへ自動連携
            ↓
     案件別損益計算
      売上
    - 自社人件費（稼働時間 × 日当）  ← 日報から自動取得
    - 外注費                          ← 経営管理から取得
    - 移動経費（高速・駐車場）        ← 日報から自動取得
    - その他経費                      ← 経営管理から取得
    = 実粗利
```

---

## 経営管理システムに追加すべき項目

### ① 案件マスタ（最優先）

**なぜ必要か：**
日報の「現場名」と経営管理の「案件名」が自由記述のため、
同じ現場が別名で登録されてデータが分断される。
共通の案件IDで両システムを紐付ける必要がある。

**追加すべき内容：**

| 項目 | 内容 |
|------|------|
| 案件ID | 例：2026-001（年-連番） |
| 案件名 | 正式名称（両システムで統一） |
| 得意先名 | 請求先（日報の請求集計と一致させる） |
| 契約金額（税抜） | 見積・契約ベース |
| 開始日・完了予定日 | 工期管理用 |
| ステータス | 進行中 / 完了 / 請求済 / 入金済 |

**LINEコマンド例（追加）：**
```
案件登録
案件名：△△ビル電気工事
得意先：〇〇社
金額：500000
開始：2026/05/01
```

---

### ② 従業員マスタ（最優先）

**なぜ必要か：**
日報に稼働時間・人員名は記録されているが、
人件費（時給 or 日当）の設定がどこにもないため、
人件費計算が不可能な状態。

**追加すべき内容：**

| 項目 | 内容 |
|------|------|
| 氏名 | 日報システムのEMPLOYEE_MAPと一致させる |
| LINE UserID | 日報システムと共通 |
| 雇用形態 | 正社員 / 日雇い / 外注 |
| 日当 or 時給 | 人件費計算に使用 |
| 備考 | 社会保険・交通費支給有無など |

**現状のEMPLOYEE_MAP（日報システム）：**
```
野添優   → 正社員（代表）
丸田翔吾 → 要設定
源地健史 → 要設定
服部秀一 → 要設定
梶原通信 → 外注扱い（別途請求あり）
```

---

### ③ 移動費の自動連携

**なぜ必要か：**
日報システムは高速・距離・駐車場をすでに記録している。
これを経営管理システムの経費として自動連携すれば二重入力が不要になる。

**対応方針：**
- 日報保存時に移動費（高速・駐車場）を経費管理Excelの「経費」シートへ自動追記
- 距離（km）は後で燃料費計算に使えるよう保持する
- 案件IDで紐付けることで案件別移動コストが自動集計できる

**経費シートに追加すべき列：**
```
既存：日付・種類・支払方法・店名・案件・金額・備考
追加：← 案件ID（日報との紐付けキー）
```

---

### ④ 案件別損益シート（Excelに追加）

**なぜ必要か：**
現状の経費管理Excelには「売上管理」「外注管理」「経費」の3シートがあるが、
案件ごとに損益をまとめるシートがない。

**追加するシート：案件損益**

| 列名 | データソース |
|------|------------|
| 案件ID | 案件マスタ |
| 案件名 | 案件マスタ |
| 得意先 | 案件マスタ |
| 売上（税抜） | 売上管理シート |
| 外注費合計 | 外注管理シート |
| 人件費合計 | 日報の稼働時間 × 日当 |
| 移動費合計 | 経費シート（種類=移動費） |
| その他経費 | 経費シート（その他） |
| 実粗利 | 売上 - 外注 - 人件費 - 経費 |
| 粗利率 | 実粗利 ÷ 売上 |

---

### ⑤ LINEの照会コマンド追加

連携後に追加すべき照会コマンド：

| コマンド | 内容 |
|----------|------|
| △△工事の損益 | 案件別の実粗利を表示 |
| 今月の人件費は？ | 日報の稼働時間から算出した人件費合計 |
| 〇〇の今月稼働時間 | 人員別の稼働時間合計 |
| 案件一覧 | 進行中の案件リスト |
| 今月の移動費は？ | 日報から自動連携した移動費合計 |

---

### ⑥ 月次レポートの拡張

現状の月次レポート（毎月1日）に以下を追加：

```
【現状】
・売上合計
・入金合計
・外注費合計
・経費合計
・粗利（= 売上 - 外注 - 経費 ※人件費除く）

【追加後】
・上記に加えて：
・人員別稼働時間
・人件費合計（稼働時間 × 日当）
・真の実粗利（人件費込み）
・案件別損益トップ3（利益・損失）
```

## 実装優先度

| 優先度 | 項目 | 理由 |
|--------|------|------|
| ★★★ | 案件マスタの設計・運用ルール決定 | 両システムを繋ぐ根幹。後から変えるとデータが壊れる |
| ★★★ | 従業員マスタ（日当設定） | 人件費計算の前提。これがないと実粗利が出ない |
| ★★☆ | 移動費の自動連携 | 日報データがすでにあるので二重入力をなくせる |
| ★★☆ | 案件損益シートの追加 | 上記2つが揃えば自動計算できる |
| ★☆☆ | LINEコマンド追加 | 案件損益シートが完成してから |
| ★☆☆ | 月次レポート拡張 | 最後で良い |

## 設計上の注意点

### 案件名の表記統一が最重要
日報の「現場名」と経営管理の「案件名」が少しでも違うと連携できない。
**案件IDを先に決めて、両システムでIDを使う運用にする**のが安全。

### 梶原通信の扱い
日報システムではEMPLOYEE_MAPに含まれているが外注扱い。
経営管理システムでは「外注請求」として処理する。
稼働時間は日報に記録されるが、人件費計算には含めず外注費として管理する。

### 月次ファイル設計の見直し検討
現状：日報（月別）・経費管理（月別）が別ファイル
連携後：案件は月をまたぐため、案件損益は月別ファイルに入れると集計が複雑になる。
→ 案件損益だけは「年間ファイル」1本にまとめることを推奨。
